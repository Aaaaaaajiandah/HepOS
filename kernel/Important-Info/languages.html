Languages for kernel development

Operating systems are usually written in a mix of low-level languages. The best choice depends on safety, performance, portability, and the amount of manual control required.

C

C is the classic choice for kernels because it offers excellent performance, predictable behavior, and a long history of use in operating systems. It is still one of the most practical choices for portable low-level code, especially when a project needs straightforward system-level access.

C++

C++ can be used for kernel development when a project wants stronger abstractions, object-oriented structure, or templates. It is powerful, but its complexity and the need for careful memory and exception handling make it a less common choice for bare-metal kernels.

Rust

Rust is attractive for kernels because it can provide memory safety and concurrency safety without a garbage collector. That makes it a strong option for modern systems software, especially when the goal is to reduce whole classes of bugs while still remaining low-level.

Assembly

Assembly is still essential for the lowest-level boot code, interrupt entry points, context switches, and architecture-specific glue. It is not usually used for an entire kernel unless the project is very small or wants maximum control over a specific CPU feature.

Zig

Zig is a newer systems language that offers a simpler and more explicit model than C or C++, with good interop and low-level control. It is appealing for kernel work because it can be more ergonomic than C while still being close to the hardware.

What HepOS uses

HepOS is primarily written in Rust, with the required low-level assembly pieces handled where the architecture demands them. This keeps the overall system modern and safer while still allowing direct control over boot and hardware interaction.
