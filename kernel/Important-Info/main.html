HepOS

A custom operating system built from scratch in Rust, with a graphical desktop, a custom filesystem, and a boot experience powered by Limine. HepOS is designed as an exokernel: the kernel handles hardware, while the rest of the system lives in a growing kernel-space libOS.

What HepOS is

HepOS is a small but ambitious x86_64 operating system that pushes the boundaries of a bare-metal Rust kernel. It includes features such as a framebuffer-based desktop, a terminal shell, a text editor, a custom filesystem, PCI and NVMe support, and early networking work. The project is still evolving, but the core kernel infrastructure is already in place.

Highlights

Desktop and window manager

A floating compositor with drag-to-move and drag-to-resize windows, a taskbar, and a start menu for launching apps.

Terminal shell

More than 30 built-in commands, history, cursor movement, tab completion, and basic filesystem operations.

Text editor and file browser

Open files from the shell or the HepFS navigator, edit text, search, save, and navigate directories.

Custom storage stack

HepFS uses a flat-inode layout on NVMe with direct and indirect blocks, plus a kernel manifest at /kernel.txt.

Drivers and hardware support

  • PS/2 keyboard
  • NVMe storage
  • XHCI USB tablet mouse
  • RTL8139 and e1000 networking

Kernel infrastructure

  • GDT and IDT
  • PMM and paging
  • x2APIC scheduler
  • Serial debug and panic reporting

Architecture at a glance

LibOS (kernel space) — desktop, shell, editor, HepFS navigator, and app logic.
Kernel — memory management, scheduling, interrupts, ACPI, and core abstractions.
Hardware — display, storage, input, networking, and PCI devices.

Build and run

HepOS builds on Windows and Linux with scripts that compile the kernel, generate the ISO, and launch QEMU.

Windows: .\build.ps1
Linux: ./build.sh

The project targets x86_64-unknown-none and uses the Limine bootloader with a bootable ISO and NVMe disk image.

Roadmap and next milestones

Near-term goals

  • Syscall interface and ring-3 userspace
  • ELF loader for user programs
  • Basic std shim for Rust crates

Platform work

  • Audio support with Intel HDA
  • Double-indirect blocks for larger files
  • Improved networking and real-world validation

Known issues

  • Networking RX is still flaky in some QEMU setups
  • Memory freeing is limited by the current bump allocator
  • ACPI shutdown still needs more hardware-aware work
